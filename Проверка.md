## Что делает команда

- Показывает общую информацию по файлу:
  - имя файла, путь, размер, количество элементов.
- При желании пользователя находит **топ-5 самых "тяжёлых" семейств** по размеру RFA-файла.
- Проверяет:
  - элементы в рабочих наборах `00_*` (кроме DWG-наборов) — там должны лежать только Revit-связи;
  - что все DWG находятся в рабочих наборах, в имени которых есть `DWG`;
  - дублированные Revit-связи с одинаковым именем (типом);
  - незакреплённые оси, уровни и связи.
- Дополнительно (кроме моделей АР/AR, КР/KR, КМ/KM):
  - ищет семейства, которые **могут не учитываться в спецификациях**
    (пустой или нулевой `ADSK_Количество`);
  - ищет семейства, которые **может быть не из библиотеки**
    (в имени семейства нет `_`).
- Ищет **дублированные общие параметры** с одинаковым названием.

Все результаты выводятся в окно отчёта pyRevit с кликабельными ссылками на элементы.

---

## Как пользоваться

1. Нажмите кнопку скрипта на панели **pyRevit**.
2. В появившемся окне ответьте на вопрос:
   - «Проверять размер загружаемых семейств?»  
   - **Да** — будет сформирован раздел с топ-5 самых тяжёлых семейств.  
   - **Нет** — этот раздел будет пропущен, проверка пойдёт дальше.
3. Дождитесь завершения проверки:
   - в окне вывода pyRevit появится структурированный отчёт по разделам;
   - проблемные элементы можно выделить по клику на ссылку в отчёте.

---

## Важно

- Скрипт **ничего не изменяет** в модели, он только анализирует и формирует отчёт.
- Для моделей, в названии которых есть маркеры `АР/AR`, `КР/KR`, `КМ/KM`,
  проверки спецификации (`ADSK_Количество`) и «библиотечных» имён семейств
  **автоматически отключаются**.
